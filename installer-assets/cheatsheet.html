ChessNavigator Cheat Sheet

Move Entry

New moves are entered by dragging pieces with the mouse.

Caps Lock: Toggle click-based moves (dragging disabled while Caps Lock is on).

Promotion: When a pawn reaches the final rank, use a keyboard shortcut to select the promotion piece:

r / s / b / q Choose promotion piece (Rook / Knight / Bishop / Queen)

Keyboard Shortcuts

HOME / R Return to home position
INSERT Save current position as home
Z Zero the board
1 / 2 / 3 / 0 Highlight hovered square: RED / YELLOW / GREEN / NO-HIGHLIGHT
DELETE Clear all highlighting
Ctrl + C Copy current position to clipboard as FEN
- / + Zoom out / Zoom in (adjust window size)

Navigating a Pre-defined Move Tree (via .txt file)

HOME / R Return to current home position
F1 / F3 Cycle to next / previous FEN in the loaded file
⮕ (RIGHT ARROW) Step into predefined move tree
⬅ (LEFT ARROW) Step back along predefined move tree
END Jump to end of predefined move tree

Adding / Removing Pieces

Drag pieces from the extra pieces panel to place them on the board. Using the right mouse button allows you to drop multiple identical pieces. To remove pieces, drag them off the board.

Customizations

Edit config.json in the executable folder to override default RGB board colors and start-up window size:

{
    "white_squares": [238, 238, 210],
    "black_squares": [118, 150, 86],
    "panel_colour": [20, 60, 60],
    "square_size": 70,
    "title_font_size": 28,
    "stip_font_size": 28,
    "info_font_size": 20,
    "animation_frames": 30,
    "animation_type": "overshoot",
    "animation_ghost": true,
    "animate_knight_hops": true
}

Note: Square sizes must be one of 40,50,60,70,80,90,100, as only these have anti-aliased piece images.

animation_type accepts "overshoot", "smooth" or "none". Set animation_frames to 1 for instant moves.

Command Line Options

Window / Interface

--window "New title" Specifies window title
--nomoves Disables the 'moves' navigation pop-up

Multiple FEN Files

--fenlist problems.txt Specifies a file containing multiple FEN positions (default is PROBLEM_LIST.txt).

You can use this file to define a small database of problems. Each line follows the PROBLEM_LIST.txt syntax, allowing pre-load then navigation in sequence.

Single FEN and Meta Info

--fen "8/7R/2K4k/..." Pass a single FEN position
--title "My Problem" Force a title for the single FEN
--stip "h#2" Force a stipulation for the single FEN

Fairy Pieces

Standard pieces: K Q R B S P for King, Queen, Rook, Bishop, Knight, Pawn.

Run the program once to generate a default custom_pieces.yml file, then edit it to add or modify available fairy pieces.

The FEN naming rules follow those of Popeye software:

  • Single letter, e.g. s or N
  • Dot + 2 letters/numbers, e.g. .ab, .L1, .l1
  • Optional = at front means neutral

Fairy pieces in a FEN appear in the panel for editing. Pieces that only appear during play may not appear. Use fairy_piece_blocks.json to specify pieces that come in families.

Additional families can be added if needed, though standard white/black variants are automatically included. Do not add existing standard pieces — if you have changed names or any previously defined pieces, delete piece_map.json before restarting.

Advanced Users: Piece Image Conventions

Piece images are loaded from the images/ folder using a naming convention based on colour, base_type, and square size:

  • Filename format: <prefix><letter>_<square_size>px.png
  • Prefix: w = white, b = black, n = neutral
  • Letter: Taken from the piece's base_type. For standard pieces: K=King, Q=Queen, R=Rook, B=Bishop, S=Knight, P=Pawn; for some fairy pieces: C=Camel, GI=Giraffe, Z=Zebra.
  • If you add a new piece base_type, the first letter of its name is used by default. To customize further, you will need to edit pieces.py to define the mapping.

Rotation and mirroring specified in the piece definition are applied to the base image. Ensure your image filenames match the convention for the program to locate them correctly.
